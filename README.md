# Customer Support Ticket Analytics

Analytics and dashboarding project focused on understanding customer support ticket patterns, service quality, and operational pressure points.

## Why I Built This

Support ticket data is useful for finding where customers struggle, where teams spend time, and where service processes can improve. This project was built to practise business intelligence, data quality review, and stakeholder style reporting.

## Business Questions

The project looks at ticket volume, issue categories, response patterns, priority levels, and data quality gaps that affect reliable reporting.

## Tools Used

Power BI, Tableau, Excel, data cleaning, dashboard design, and business reporting.

## What I Implemented

I reviewed the support ticket dataset, explored service patterns, identified reporting and data quality issues, and prepared dashboard assets for business review. The project also includes a written reflection report explaining findings and limitations.

## Repository Guide

The dashboards folder is for Power BI and Tableau files. The reports folder is for written analysis. The data folder is for dataset notes and safe sample files.

## Outputs And Results

The project produces a dashboard focused on support operations, a data quality finding around missing or incomplete records, and a written explanation of what the data shows.
